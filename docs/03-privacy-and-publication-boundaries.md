## 공개 불가 또는 사전 확인 필요

아래 항목은 사용자의 명시 승인 전까지 공개하지 않는다.

- 전화번호
- 실명을 메인 공개 정체성으로 사용하는 것
- 얼굴 사진
- 업무 화면 이미지
- Figma 캡처
- 내부 문서
- 기밀 소스 코드
- 고객사 내부 정보
- 검증되지 않은 성과 수치
- 프로젝트 내부 URL, 계정, 토큰, 서버 정보

## 전화번호 정책

- 공개 문서와 사이트 데이터에 전화번호를 넣지 않는다.
- 내부 구조화 이력서에는 값 대신 `DO_NOT_PUBLISH`만 사용한다.
- PDF 이력서도 공개 전에 전화번호 제거 여부를 확인한다.

## PDF 이력서 정책

사용자는 공개용 PDF 이력서를 `이혜주_FrontendEngineer.pdf`로 제공했다. PDF 내부 내용은 사용자가 공개 가능한 것으로 확인했으므로 별도 내용 검사는 요구하지 않는다.

개발 단계에서 할 일:

1. PDF 파일을 GitHub Pages에서 접근 가능한 위치에 배치한다.
2. hero와 resume section에서 다운로드 링크를 제공한다.
3. 사이트 본문의 main public identity는 계속 Peppearl로 유지한다.

## 이미지 정책

기본적으로 업무 화면과 Figma 캡처는 사용하지 않는다. 대체 표현은 아래를 우선한다.

- 추상화된 아키텍처 다이어그램
- 실제 화면을 재현하지 않는 wireframe block
- component state matrix
- decision card
- folder structure example
- 비밀 정보가 제거된 code snippet

## 강아지 사진 정책

- 얼굴 사진 대신 강아지 사진을 작은 개인적 요소로 사용할 수 있다.
- 권장 위치는 About 카드, footer, 작은 profile chip이다.
- 강아지 사진이 사이트의 주제가 되거나 hero visual을 차지하면 안 된다.

## 성과 수치 정책

구체적인 성과 수치는 공개하지 않는다. 수치가 이력서나 과거 자료에 있더라도 사이트 본문에서는 아래처럼 정성적으로 표현한다.

- 유지보수 기준을 정리했다.
- 반복 구현을 줄일 수 있는 구조를 마련했다.
- 사용자 흐름을 더 예측 가능하게 정리했다.
- 개발 환경과 타입 안정성을 개선했다.
- 복잡한 업무 데이터를 화면 구조로 정리했다.

## 코드 스니펫 정책

대표 케이스를 설명하기 위해 코드가 필요하면 먼저 사용자에게 요청한다. 사용자가 제공하기 어렵다고 답하면 실제 코드 대신 다이어그램과 pseudo code 중심으로 설명한다.

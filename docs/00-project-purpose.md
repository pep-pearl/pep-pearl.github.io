# 프로젝트 목적

이 저장소는 Peppearl의 GitHub Pages 개인 커리어 사이트를 만들기 위한 작업 공간이다. 목표는 이력서, 경력기술서, 포트폴리오를 하나의 커리어 랜딩페이지로 묶어 포지션 제안과 면접 제안을 자연스럽게 유도하는 것이다.

## 핵심 목표

- GitHub 중심 포트폴리오가 아니라, 채용 담당자와 기술 리더가 빠르게 검토할 수 있는 커리어 랜딩페이지를 만든다.
- 방문자가 3분 안에 Peppearl의 강점과 경력 흐름을 이해할 수 있어야 한다.
- PDF 이력서를 열지 않아도 경력, 주요 프로젝트, 기술 방향성이 파악되어야 한다.
- 면접관이나 채용 담당자에게 링크 하나만 보내도 되는 상태를 목표로 한다.
- 실무 프로젝트, 구조적 사고, 유지보수 가능한 프론트엔드 구현 역량을 GitHub보다 앞에 둔다.

## 방문자가 기억해야 할 메시지

> Peppearl은 지속 가능한 디자인 시스템과 확장성 있는 아키텍처로 제품의 완성도를 높이는 프론트엔드 개발자다.

이 메시지는 아래 세 가지 역량으로 반복해서 뒷받침한다.

1. Design System
2. Frontend Architecture
3. AI Agent Workflow

## 이 사이트가 아닌 것

- 일반 개발 블로그
- GitHub 활동을 전면에 둔 쇼케이스
- 토이 프로젝트 아카이브
- 과장된 자기소개 랜딩페이지
- 업무 화면 캡처 중심의 포트폴리오
- 공공/SI 프로젝트 목록처럼 보이는 페이지

## 성공 기준

- 채용 담당자가 3분 안에 역할과 강점을 이해한다.
- 프론트엔드 리드가 기술적 판단과 구조화 역량을 확인할 수 있다.
- CTO나 창업자가 복잡한 업무 시스템을 안정적으로 다룰 수 있다는 신뢰를 얻는다.
- 대표 프로젝트 3개가 스크린샷이나 과장된 수치 없이도 설득력 있게 읽힌다.
- 이력서 PDF 다운로드는 가능하지만, PDF 없이도 핵심 경력이 전달된다.
- 모바일, 태블릿, 데스크톱에서 모두 자연스럽게 읽힌다.

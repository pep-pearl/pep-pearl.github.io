# AI Agent Workflow Source

이 문서는 사용자가 추가한 vibe coding workflow 자료를 포트폴리오용으로 정리한 것이다. 실제 사이트에는 긴 프롬프트 전문을 그대로 넣지 않고, 구조와 원칙을 요약해 사용한다.

## Core Principle

AI Agent는 코드를 대신 작성하는 자동화 도구가 아니라, 사람이 검토 가능한 계획과 기준을 만드는 작업 파트너다. 모든 단계에서 사람의 검토와 승인 gate가 중심이다.

## Workflow

```txt
Deep Research
-> Strategic Planning
-> Human Review & Iteration
-> Implementation
-> Verification
```

## Phase 1. Deep Research

### 목적

관련 코드와 의존성을 깊이 분석하고, 수정 전 맥락을 문서화한다.

### 역할

- 코드베이스 구조 이해
- 관련 파일과 호출 관계 분석
- 수정 또는 기능 추가의 영향 범위 파악
- `research.md` 같은 분석 산출물 생성

### 사이트에서 보여줄 수 있는 문장

"구현 전에 관련 코드와 의존성을 먼저 분석해, 변경이 닿는 범위를 문서로 정리합니다."

## Phase 2. Strategic Planning

### 목적

실행 에이전트가 임의 판단 없이 따라갈 수 있을 정도로 구체적인 계획을 만든다.

### 포함할 내용

- 요구사항 체크리스트
- 리스크와 edge case
- 변경 지점
- pseudo code
- side effect 예상
- verification 기준
- 실패 시 중단 기준

### 사이트에서 보여줄 수 있는 문장

"계획 단계에서는 실제 코드를 수정하지 않고, 요구사항과 리스크, 변경 지점, 검증 기준을 먼저 정리합니다."

## Phase 3. Human Review & Iteration

### 목적

AI가 만든 계획을 사람이 검토하고, 필요한 피드백을 반영한 뒤 승인한다.

### 원칙

- 승인 전 코드 수정 금지
- 인라인 코멘트 반영
- 비즈니스 로직 준수 여부 확인
- 가설과 리스크 재검토

### 사이트에서 보여줄 수 있는 문장

"AI가 만든 계획을 그대로 실행하지 않고, 사람이 검토하고 수정한 뒤 다음 단계로 넘깁니다."

## Phase 4. Implementation

### 목적

승인된 계획을 기준으로 구현한다.

### 원칙

- 계획서의 검증 단계를 통과하지 못하면 중단
- 예상치 못한 side effect가 생기면 보고
- 하위 모델 또는 직접 구현은 계획의 범위 안에서만 진행

## Model Tiering

작업 성격에 따라 모델 역할을 나눈다.

| 단계 | 목적 | 모델 사용 방향 |
| --- | --- | --- |
| Deep Research | 코드와 의존성 분석 | 깊은 코드 이해가 가능한 모델 |
| Strategic Planning | 리스크와 계획 수립 | 추론과 설계에 강한 모델 |
| Human Review | 계획 검토와 수정 | 사람이 최종 판단 |
| Implementation | 승인된 계획 실행 | 구현에 적합한 모델 또는 직접 구현 |

## Portfolio Usage

### Case 03에 넣기 좋은 포인트

- Research -> Plan -> Hold -> Implement -> Review 흐름
- 계획 문서와 검토 기준을 먼저 만든다는 점
- AI 산출물에 대해 사람이 승인 gate를 둔다는 점
- 모델을 무조건 크게 쓰는 것이 아니라 작업 성격에 따라 역할을 나눈다는 점

### 피해야 할 표현

- AI가 개발을 대신한다.
- AI로 모든 문제를 해결한다.
- 자동화로 생산성이 몇 배 개선됐다.
- 사람 검토 없이 바로 구현한다.

### 사이트 카피 후보

- "AI Agent를 빠른 코드 생성보다 계획 검토와 구조 안정성을 지키는 작업 파트너로 사용합니다."
- "구현 전에 요구사항과 검토 기준을 먼저 문서화하고, 사람이 승인한 뒤 다음 단계로 넘어갑니다."
- "작업 성격에 따라 AI의 역할을 나누되, 최종 판단은 사람이 맡습니다."

# Blog Portfolio Audit

Blog: `https://jinjoo648.tistory.com/`

## 블로그 노출 가치

블로그는 사이트에 노출할 가치가 있다. 다만 메인 CTA가 아니라 보조 링크로 두어야 한다. 블로그는 실무 프로젝트의 직접 증거라기보다 학습 태도, 기술 흐름을 따라가는 방식, 문제를 정리하는 습관을 보여주는 보조 근거로 적합하다.

블로그 루트에서 확인한 카테고리는 개발 회고록, React, Linux, Tomcat, Template이며, 총 36개 글이 노출된다. 이 중 포트폴리오에 연결하기 좋은 글은 React/아키텍처 관련 글이고, Linux 보안 점검 글은 운영 맥락 이해의 보조 자료 정도로만 적합하다.

## 활용하기 좋은 섹션

- Selected Work 하단의 Related Writing
- Frontend Architecture strength card의 보조 링크
- About 하단의 "더 읽어보기"
- Footer의 Blog 링크

Hero CTA에는 넣지 않는다.

## 리스크

### 오래된 글

2025년 React/개발 회고 글은 현재 역량처럼 과장하지 않고, 학습 과정과 사고방식의 근거로만 사용한다.

### 기술 설명 중심 글

React 19, Linux 명령어 글처럼 설명 중심인 글은 사이트 본문에 길게 끌어오면 산만해진다. 링크 또는 짧은 related writing으로만 사용한다.

### 포지셔닝과 거리 있는 글

Template, 다이어리 PDF, 순수 Linux 명령어 정리 글은 커리어 랜딩페이지의 핵심 메시지를 흐릴 수 있다.

## 공개 링크로 연결해도 되는 글 기준

Include:

- React / TypeScript / 프론트엔드 구조와 직접 연결된다.
- 아키텍처 선택, 유지보수, 경계 설정을 다룬다.
- 현재 포지셔닝의 보조 근거로 읽힌다.

Maybe:

- 학습 태도는 보여주지만 오래되었거나 기술 설명 중심이다.
- 운영/보안 맥락을 보여주지만 프론트엔드 포지셔닝과 거리가 있다.

Exclude:

- 개인 템플릿 배포 글
- 너무 사적인 글
- 현재 사이트의 전문성을 흐릴 수 있는 글
- 실제로 확인하지 못한 글

## 블로그를 보조 링크로 둘 방법

- Hero primary CTA에는 넣지 않는다.
- Hero secondary links에 Blog를 작게 노출하거나 Contact/Footer에 배치한다.
- Selected Work 아래에는 관련 글 1-2개만 연결한다.
- GitHub와 Blog 모두 secondary group에 두되, 업무 경험과 Selected Work가 먼저 보이게 한다.

## 블로그 접근 관련 메모

- `FE 아키텍처를 고민하며`는 블로그 메인 페이지에서 본문을 확인했다.
- 개별 URL 직접 접근은 추가 확인이 필요하다.
- 실제로 열람하지 못한 글은 후보로 만들지 않았다.
